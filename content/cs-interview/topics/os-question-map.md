# OS Question Map

## 한 줄 정의

운영체제가 프로세스·스레드·메모리·파일·동시성을 관리하는 방식을 묻는 질문 지도.

## 카드 요약

- **프로세스/스레드**: 실행 단위, 상태, PCB, Context Switching.
- **동시성**: IPC, 경쟁 상태, 데드락.
- **메모리/파일**: 주소 공간, 세그먼테이션, 파일시스템.
- **커널 인터페이스**: 시스템 콜과 라이브러리 콜.

## 핵심 질문

- 시스템 프로그래밍
- 운영체제
- 프로그램
- 프로세스와 스레드
- 세그먼테이션
- 파일시스템
- 메모리의 주소 공간
- 시스템 콜
- 라이브러리 콜
- PCB
- Context Switching
- IPC(InterProcess Communication)
- 프로세스 상태(Process State)
- CPU 스케줄링
- 데드락
- 경쟁 상태

## 면접 포인트

1. **프로그램과 프로세스는 다르다**: 프로그램은 파일, 프로세스는 실행 중인 인스턴스다.
2. **스레드는 자원을 공유한다**: 공유 때문에 가볍지만 경쟁 상태를 조심해야 한다.
3. **Context Switching은 공짜가 아니다**: 레지스터/메모리 맵 전환과 캐시 영향까지 비용으로 본다.

- “시스템 콜과 라이브러리 콜은 어떻게 다른가요?”
