# Security Question Map

## 한 줄 정의

웹과 시스템에서 공격이 어떤 방식으로 들어오고, 암호화·인증·방어 전략으로 어떻게 줄이는지 묻는 보안 질문 지도.

## 카드 요약

- **웹 취약점**: XSS, CSRF, SQL Injection.
- **계정 공격**: 크레덴셜 스터핑.
- **네트워크 공격**: 스푸핑, 스누핑, 스니핑, DDoS, 패킷 하이재킹.
- **암호화**: HTTPS, Diffie-Hellman, RSA, AES, SHA.

## 핵심 질문

- XSS
- CSRF
- SQL Injection
- 크레덴셜 스터핑
- 스푸핑

- 백도어

- RSA 암호화
- AES
- 단방향 암호화
- 양방향 암호화
- SHA

## 면접 포인트

1. **공격 벡터와 방어책을 짝지어 말하기**: XSS는 입력 검증/이스케이프/CSP, CSRF는 SameSite/CSRF 토큰처럼 연결한다.
2. **암호화는 용도가 다르다**: AES는 대칭키, RSA는 공개키, SHA는 해시다.
3. **HTTPS는 “암호화된 HTTP” 이상**: 인증서, 키 교환, 무결성 검증까지 포함한다.

## 질문 예시

- “HTTPS 연결에서 인증서는 어떤 역할을 하나요?”
