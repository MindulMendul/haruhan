# Event Loop (Call Stack & Task Queue)

## 한 줄 정의

JS는 **단일 스레드**로 **Call Stack**을 하나만 쓰고, **비동기 콜백**은 **태스크 큐**(매크로/마이크로)에 쌓였다가 스택이 비면 실행된다.

## 카드 요약

- **Call Stack**: 동기 코드가 쌓였다 빠짐. 블로킹되면 UI 멈춤.
- **Web API**: `setTimeout`, `fetch`, DOM 이벤트 등 — 브라우저/런이 처리 후 콜백을 큐에 넣음.
- **마이크로태스크**: `Promise.then`, `queueMicrotask` — **한 매크로태스크 끝난 뒤** 전부 처리.
- **매크로태스크**: `setTimeout`, I/O 등 — 마이크로가 비면 다음 매크로 하나.
- **렌더 타이밍**: 브라우저는 보통 **매크로 사이**에 페인트 기회.

## 면접 포인트

1. **출력 순서**: 동기 → 마이크로 → (렌더 가능) → 다음 매크로.
2. **starvation**: 마이크로태스크만 무한히 넣으면 매크로·렌더가 굶을 수 있음.
3. **Node vs Browser**: 둘 다 이벤트 루프 개념은 유사, Phase 이름·세부는 다름.
