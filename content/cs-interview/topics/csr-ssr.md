# CSR & SSR (Client / Server Side Rendering)

## 한 줄 정의

**CSR**: 브라우저에서 JS로 DOM을 그림. **SSR**: 서버에서 HTML을 만들어 내려보냄.

## 카드 요약

| | CSR | SSR |
|--|-----|-----|
| **첫 화면** | JS 다운로드·실행 후 | HTML에 콘텐츠 포함 가능 |
| **SEO** | 추가 대응 필요 | 유리한 편 |
| **서버 부하** | 낮음 | 요청마다 렌더 비용 |
| **인터랙션** | 이후 전환은 빠름 | TTFB·하이드레이션 고려 |

- **SSG/ISR**: 빌드·주기적 재생성으로 SSR 비용 완화.
- **하이드레이션**: 서버 HTML + 클라이언트 React가 이벤트 연결.

## 면접 포인트

1. **언제 CSR?** — 대시보드 등 SEO 불필요, 인터랙션 중심.
2. **언제 SSR/SSG?** — 마케팅·문서·상품 상세 등 크롤링·FCP 중요.
3. **Streaming SSR**: HTML을 조각으로 보내 TTFB·LCP 개선.
