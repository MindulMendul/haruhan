# JIT Question Map

## 한 줄 정의

JIT(Just-In-Time) 컴파일러가 런타임에 코드를 최적화해 인터프리터와 정적 컴파일 사이의 균형을 잡는 방식.

## 카드 요약

- **인터프리터**: 코드를 한 줄씩 해석해 실행한다.
- **AOT 컴파일**: 실행 전에 기계어로 컴파일한다.
- **JIT 컴파일**: 실행 중 자주 쓰이는 코드를 기계어로 바꾸고 최적화한다.
- **트레이드오프**: 워밍업 비용은 있지만 반복 실행 구간에서 성능 이득이 크다.

## 핵심 질문

- JIT(Just-In-Time) 컴파일러

## 면접 포인트

1. **JIT은 “실행 중 최적화”가 핵심**: 실제 실행 패턴을 보고 뜨거운 경로를 최적화한다.
2. **JavaScript 엔진과 연결된다**: V8 같은 엔진은 인터프리터와 JIT을 조합한다.
3. **최적화는 되돌릴 수 있다**: 가정이 깨지면 deoptimization이 발생할 수 있다.

- “JIT의 단점은 무엇인가요?”
