# C++ Question Map

## 한 줄 정의

C++의 메모리 관리, 표준 라이브러리, 컨테이너 내부 동작을 묻는 면접 질문 지도.

## 카드 요약

- **메모리 초기화**: `memset`, `fill`, `fill_n`의 차이.
- **네임스페이스**: `using namespace std`와 `std::` 사용 방식.
- **동적 할당**: `new`/`delete` 짝, 스마트 포인터.
- **컨테이너/정렬**: vector capacity, `std::sort`, `list.sort`, stable sort.

## 핵심 질문

- C++ `memset()` vs `fill()`, `fill_n()`
- `using namespace std`와 `std::`의 차이점
- `new`/`delete` 쌍이 안 맞으면 생기는 일
- 쌍이 안 맞는 걸 방지할 방법
- Smart pointer를 써봤는지, 종류는 무엇인지
- `vector`의 capacity가 넘어가면 어떤 내부 동작이 있는지
- `std::sort`와 `list.sort`가 왜 구분되어 있는지
- 둘의 정렬 알고리즘은 무엇인지
- `stable_sort`가 무엇인지
- 링크드리스트와 배열의 접근 속도가 차이 나는 이유
- 1부터 1,000,000까지의 수를 무작위로 중복 없이 클라이언트에게 던져주는 알고리즘

## 면접 포인트

1. **RAII로 메모리 안전성을 설명하기**: 직접 해제보다 소유권을 객체 생명주기에 묶는 관점이 중요하다.
2. **컨테이너는 내부 구조가 성능을 결정한다**: vector는 연속 메모리, list는 노드 기반 구조다.
3. **정렬 안정성은 요구사항이다**: 같은 키의 상대 순서가 유지되어야 하면 stable sort가 필요하다.

## 질문 예시

- “스마트 포인터에는 어떤 종류가 있고 언제 쓰나요?”
- “vector가 재할당될 때 어떤 일이 생기나요?”
- “배열이 링크드리스트보다 캐시 친화적인 이유는 무엇인가요?”
