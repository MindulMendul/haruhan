# State Management

## 한 줄 정의

**UI가 보여줄 데이터**와 **그 변화의 규칙**을 어디에 두고 어떻게 전파할지 정하는 일.

## 카드 요약

- **로컬 상태**: `useState`, 컴포넌트 안에서만 — 범위 최소화가 기본.
- **서버 상태**: React Query, SWR — **캐시·재검증·동기화**가 핵심(“전역 스토어”와 다름).
- **URL 상태**: 필터·페이지·모달 쿼리 — 공유·뒤로가기에 유리.
- **전역 클라이언트 상태**: Redux, Zustand, Jotai — **여러 화면이 같은 소스**가 필요할 때.
- **선택 기준**:  
  - 서버 데이터 → 캐시 라이브러리  
  - 폼·애니메이션 → 로컬  
  - 테마·유저 설정 → 컨텍스트/경량 스토어

## 면접 포인트

1. **왜 전역을 줄이나?** — 불필요한 리렌더·결합도·테스트 어려움.
2. **Redux 3원칙**: 단일 스토어, 읽기 전용 상태, 순수 reducer.
3. **React Query + Zustand** 같은 **역할 분리**가 요즘 흔한 조합.
