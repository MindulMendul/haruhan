# Functional Programming & Prototype

## 한 줄 정의

**함수형**: 계산을 **함수 합성**과 **불변 데이터**로 표현. **프로토타입**: JS 객체가 다른 객체에서 **속성·메서드를 상속**하는 메커니즘.

## 카드 요약

### 함수형 프로그래밍

- **순수 함수**: 같은 입력 → 같은 출력, 부수 효과 없음.
- **불변성**: 상태를 바꾸지 않고 새 값을 만듦(예: `map`, `filter`).
- **고차 함수**: 함수를 인자로 받거나 반환.
- **React 연결**: `useState`의 setter, `useReducer`, 불변 업데이트로 리렌더 최적화.

### Prototype

- `__proto__`(또는 `Object.getPrototypeOf`)로 체인 연결.
- `new` + `constructor` → `prototype` 객체에 메서드.
- **클래스 문법**은 프로토타입 위의 설탕.
- **`extends`**도 프로토타입 체인으로 구현.

## 면접 포인트

1. **왜 불변성?** — 참조 비교·예측 가능한 리렌더·시간 여행 디버깅.
2. **`this` 바인딩**과 프로토타입: 화살표 함수는 `this`를 가지지 않음.
3. **FP vs OOP**: React는 FP 스타일 선호, 클래스는 레거시·일부 라이브러리.
