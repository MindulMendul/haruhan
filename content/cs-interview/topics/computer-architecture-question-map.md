# Computer Architecture Question Map

## 한 줄 정의

컴퓨터가 명령을 실행하고 데이터를 표현·저장·검증하는 방식을 묻는 컴퓨터 구조 질문 지도.

## 카드 요약

- **하드웨어 구성**: CPU, 메모리, 입출력 장치의 역할.
- **메모리 계층**: 캐시 메모리와 가상 메모리.
- **데이터 표현**: 실수 표현, 패리티 코드, 해밍 코드.
- **압축/명령어**: 허프만 코드, 명령어 집합.

## 핵심 질문

- 컴퓨터의 구성
- 중앙처리장치(CPU)
- 캐시 메모리
- 가상 메모리
- 컴퓨터의 실수 표현
- 패리티 코드
- 해밍 코드
- 허프만 코드
- 명령어 집합

## 면접 포인트

1. **성능은 메모리 접근 비용에서 갈린다**: CPU가 빠를수록 캐시 지역성 설명이 중요하다.
2. **가상 메모리는 “큰 메모리처럼 보이게 하는 기술” 이상**: 보호, 주소 변환, 페이지 교체까지 연결된다.
3. **오류 검출/수정은 목적이 다르다**: 패리티는 검출 중심, 해밍은 일부 오류 수정까지 가능하다.

## 질문 예시

- “캐시 메모리는 왜 필요한가요?”
- “가상 메모리와 물리 메모리는 어떻게 다르나요?”
- “실수 연산에서 오차가 생기는 이유는 무엇인가요?”
