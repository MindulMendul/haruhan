# Algorithm Question Map

## 한 줄 정의

문제를 어떤 절차로 풀고, 시간·공간 복잡도를 어떻게 줄이는지 묻는 알고리즘 질문 지도.

## 카드 요약

- **기본 전략**: 정렬, 탐색, 그리디, 동적 계획법.
- **그래프/집합**: 그래프, 유니온 파인드.
- **구간/문자열**: 문자열 탐색, 투 포인터, 슬라이딩 윈도우, 스위핑, 누적합.
- **수학/완전탐색**: 조합론, 백트래킹, 소수 판정, 오일러 피 함수.

- 알고리즘
- 정렬
- 탐색
- 동적 계획법(다이나믹 프로그래밍)
- 그리디 알고리즘
- 그래프
- 문자열 탐색
- 비트마스크
- 투 포인터
- 슬라이딩 윈도우
- 스위핑
- 유니온 파인드
- 조합론
- 백트래킹

- 오일러 피 함수
- 유명한 알고리즘 문제

## 면접 포인트

1. **복잡도부터 말하기**: 풀이 선택 이유를 `O(n)`, `O(n log n)`, `O(n^2)` 같은 비용으로 설명한다.
2. **DP는 상태 정의가 핵심**: 점화식보다 먼저 “무엇을 저장할 것인가”를 잡는다.
3. **그리디는 증명이 필요하다**: 매 순간 최선이 전체 최선으로 이어지는 이유를 말해야 한다.

## 질문 예시

- “DP와 그리디는 어떻게 구분하나요?”
- “투 포인터와 슬라이딩 윈도우는 언제 쓰나요?”
- “유니온 파인드는 어떤 문제에서 유용한가요?”
