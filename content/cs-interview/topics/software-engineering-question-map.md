# Software Engineering Question Map

## 한 줄 정의

코드를 오래 유지하고 팀이 안정적으로 배포하기 위해 필요한 설계·프로세스·품질 질문 지도.

## 카드 요약

- **코드 품질**: 클린코드, 리팩토링, 시큐어 코딩.
- **개발 방법론**: 애자일, TDD.
- **패러다임/설계**: 객체 지향, 함수형 프로그래밍, 디자인 패턴.
- **운영/아키텍처**: DevOps, Third Party, MSA, CI/CD.

## 핵심 질문

- 클린코드
- 리팩토링
- 시큐어 코딩
- 애자일
- 객체 지향 프로그래밍
- 함수형 프로그래밍(Functional Programming)
- 데브옵스(DevOps)
- 서드 파티(Third Party)
- 마이크로서비스 아키텍처(MSA, MicroService Architecture)
- 디자인 패턴
- CI/CD
- TDD

## 면접 포인트

1. **좋은 코드는 변경 비용이 낮다**: 읽기 쉬움, 테스트 가능성, 의존성 방향을 같이 설명한다.
2. **리팩토링은 기능 변경이 아니다**: 외부 동작은 유지하면서 내부 구조를 개선한다.
3. **CI/CD는 자동화 이상의 문화**: 작은 변경, 빠른 피드백, 실패 시 롤백 전략이 중요하다.

## 질문 예시

- “클린코드를 어떻게 정의하나요?”
- “TDD의 장단점은 무엇인가요?”
- “MSA를 선택하면 어떤 비용이 생기나요?”
